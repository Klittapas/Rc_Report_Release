Figure 2

Class Diagram

โครงสร้างข้อมูล (types) + โมดูล logic โปรแกรมเป็น functional — คลาส OOP จริงมีแค่ Interner, logic ส่วนอื่นแสดงเป็น «module»

%%{init:{'theme':'neutral'}}%%
classDiagram
  class normalize {
    <<module>>
    +normalizeCsv(input) NormalizeResult
    -parseCSV(text) string[][]
    -decode(code) [ch, plan, board, refund]
    -promoCategory(typ, refund) string
    -boardOf(code) string
    -toISO(stay) string
  }
  class Interner {
    +list : string[]
    -map : Map~string, number~
    +idx(v) number
  }
  class NormalizeResult {
    +dataset : Dataset
    +rowsRead : number
    +rowsKept : number
    +skippedTotals : number
  }
  class Dataset {
    +generated : string
    +hotels : string[]
    +segments : string[]
    +plans : string[]
    +channels : string[]
    +roomTypes : string[]
    +boards : string[]
    +refunds : string[]
    +dates : string[]
    +rows : number[][]
  }
  class aggregate {
    <<module>>
    +aggregate(ds, f) HotelAgg[]
  }
  class Filters {
    +segments : Set~string~
    +startIdx : number
    +endIdx : number
  }
  class HotelAgg {
    +name : string
    +total_revenue : number
    +total_rooms : number
    +adr : number
    +best_promo : string
    +promos : Promo[]
    +channels : Channel[]
    +trend : TrendPt[]
  }
  class cluster {
    <<module>>
    +standardize(data) z, means, sds
    +kmeans(z, k) ClusterResult
  }
  class ClusterResult {
    +assignments : number[]
    +centroids : number[][]
    +inertia : number
  }
  normalize ..> Interner : uses
  normalize ..> NormalizeResult : returns
  NormalizeResult *-- Dataset
  aggregate ..> Dataset : reads
  aggregate ..> Filters : reads
  aggregate ..> HotelAgg : returns
  cluster ..> ClusterResult : returns
ปรับปรุง: เพิ่มโมดูล cluster (k-means จัดกลุ่ม day-type) + ClusterResult ที่เพิ่งเพิ่มเข้าโปรเจกต์ ยังไม่มีใน UML เดิม

Sequence Diagram — Upload & Render

ลำดับเหตุการณ์ตั้งแต่ผู้ใช้เลือกไฟล์ CSV จนกราฟขึ้นจอ ไฟล์ไม่ถูกส่งออกนอกเครื่อง

%%{init:{'theme':'neutral'}}%%
sequenceDiagram
  actor User
  participant App as App.tsx
  participant N as normalize.ts
  participant A as aggregate.ts
  participant S as sections/*
  participant C as Chart.js
  User->>App: เลือกไฟล์ CSV (อ่านเป็น text)
  App->>N: normalizeCsv(texts[])
  N->>N: parseCSV (RFC-4180)
  N->>N: decode(rate code)
  N->>N: intern + accumulate rows
  N-->>App: NormalizeResult { dataset, rowsRead, rowsKept }
  App->>A: aggregate(dataset, filters)
  A-->>App: HotelAgg[]
  App->>S: render sections (props = HotelAgg)
  S->>C: draw charts
  C-->>User: แสดงกราฟบนหน้าจอ
Figure 4

Activity Diagram — normalizeCsv()

ตรรกะแกนของโปรเจกต์ แปลงรหัสราคาดิบ (เช่น bdcmnmnr) เป็น Channel + Promotion + Board + Refund และรวมยอด (รวมแถวยกเลิกค่าติดลบ)

%%{init:{'theme':'neutral'}}%%
flowchart TB
  START([Start]) --> P[parseCSV แต่ละไฟล์]
  P --> H{เจอ header row
Stay/Booking Date?}
  H -- ไม่เจอ --> ERR[throw: format ผิด]
  H -- เจอ --> MAP[map คอลัมน์ตามชื่อ header]
  MAP --> SEG{มีคอลัมน์
Market Segment?}
  SEG -- ไม่มี --> ERR2[throw: หา Segment ไม่เจอ]
  SEG -- มี --> LOOP{อ่านทีละแถว}
  LOOP -- แถว Total --> SKIP[ข้าม subtotal]
  SKIP --> LOOP
  LOOP -- แถวข้อมูล --> EMPTY{rooms=0 และ rev=0?}
  EMPTY -- ใช่ --> LOOP
  EMPTY -- ไม่ --> DEC[decode rate code
Channel/Plan/Board/Refund]
  DEC --> ISO[toISO วันที่]
  ISO --> ACC[intern + สะสมเข้า acc
บวกยอด รวมค่าติดลบ]
  ACC --> LOOP
  LOOP -- หมดแถว --> BUILD[สร้าง rows number matrix]
  BUILD --> OUT([return Dataset])
หมายเหตุ: ตรงกับโค้ดปัจจุบัน — Market Segment ยัง บังคับ (throw ถ้าไม่มี ที่ normalize.ts:144) หากภายหลังทำ segment ให้ optional + เพิ่ม Stay Date ต้องแก้แผนภาพนี้

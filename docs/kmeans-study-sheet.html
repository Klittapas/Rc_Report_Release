1 ภาพรวม 30 วินาที

K-means รับข้อมูล n จุด แล้วแบ่งเป็น k กลุ่ม โดยพยายามให้จุดในกลุ่มเดียวกันอยู่ใกล้กันที่สุด เป็น unsupervised — ไม่มีเฉลย เรียนจากความเหมือนของข้อมูลล้วนๆ

ป้อนเข้า (Input)

เมทริกซ์ X ขนาด n×d
(n จุด, แต่ละจุด d feature) และจำนวนกลุ่ม k

ได้ออก (Output)

label ให้แต่ละจุด (อยู่กลุ่มไหน) + centroid k จุด (ศูนย์กลางแต่ละกลุ่ม)

2 สัญกรณ์ & สูตรหลัก

Objective — สิ่งที่ k-means ย่อให้เล็กสุด
J = Σi=1n ‖ xi − μc(i) ‖²
xi = จุดที่ i · μc(i) = centroid ของกลุ่มที่จุด i สังกัด · J เรียกว่า inertia (ผลรวมระยะยกกำลังสองในกลุ่ม)
ระยะทาง Euclidean (ยกกำลังสอง)
d²(a, b) = Σj=1d (aj − bj)²
ไม่ถอดรูทเพื่อความเร็ว — การเปรียบเทียบว่าอันไหนใกล้กว่าให้ผลเท่ากัน
Centroid update — จุดศูนย์กลางใหม่
μc = (1 / |Cc|) · Σi∈Cc xi
= ค่าเฉลี่ยของสมาชิกในกลุ่ม Cc (ที่มาของชื่อ "mean")
z-score — ทำให้ทุก feature สเกลเดียวกัน (สำคัญมาก)
zj = (xj − μj) / σj
μj, σj = ค่าเฉลี่ย/SD ของ feature ที่ j · ทำก่อนเข้า k-means เสมอ
Silhouette — วัดว่ากลุ่มชัดแค่ไหน (−1 ถึง 1)
s(i) = (bi − ai) / max(ai, bi)
ai = ระยะเฉลี่ยไปเพื่อนร่วมกลุ่ม · bi = ระยะเฉลี่ยไปกลุ่มข้างบ้านที่ใกล้สุด · คะแนนรวม = เฉลี่ย s(i) ทั้งชุด

3 อัลกอริทึม (Lloyd's algorithm)

  1. Standardize — z-score ทุกคอลัมน์ ให้ feature เสียงเท่ากัน
  2. Init — วาง centroid เริ่มต้น k จุด (ใช้ k-means++ ดีกว่าสุ่มมั่ว)
  3. Assign — จุดทุกจุดเข้าหา centroid ที่ใกล้ที่สุด (ใช้ d²)
  4. Update — เลื่อน centroid แต่ละตัวไปที่ค่าเฉลี่ยของสมาชิก
  5. วนซ้ำ 3→4 จนไม่มีจุดไหนเปลี่ยนกลุ่ม = ลู่เข้า (converged)
  6. Restart ทั้งหมดหลายรอบ (เช่น 12) เก็บรอบที่ inertia ต่ำสุด
ทำไมหยุดแน่: ทุกจังหวะ assign และ update มีแต่ทำให้ J (inertia) ลดลงหรือเท่าเดิม ไม่มีทางเพิ่ม → รับประกันลู่เข้า แต่ไม่รับประกันว่าดีที่สุด (จึงต้อง restart)

4 โค้ด Python — หัวใจทีละส่วน

4.1 Standardize (z-score)

def standardize(X):
    mean = X.mean(axis=0)
    sd = X.std(axis=0)
    sd[sd == 0] = 1.0          # กันหารศูนย์
    return (X - mean) / sd, mean, sd

4.2 k-means++ seeding

def _kpp_init(Z, k, rng):
    centroids = [Z[rng.integers(len(Z))]]
    for _ in range(1, k):
        # ระยะ² ไป centroid ใกล้สุดที่มีอยู่แล้ว
        d2 = np.min([((Z - c)**2).sum(1) for c in centroids], axis=0)
        probs = d2 / d2.sum()             # ไกล = โอกาสถูกเลือกสูง
        centroids.append(Z[rng.choice(len(Z), p=probs)])
    return np.array(centroids)

4.3 ลูปหลัก assign / update

for _ in range(max_iter):
    # ASSIGN: dists[i,c] = ระยะ² จุด i ถึง centroid c
    dists = ((Z[:, None, :] - C[None, :, :])**2).sum(2)
    new_labels = dists.argmin(1)
    if np.array_equal(new_labels, labels):
        break                             # ไม่มีใครย้าย = จบ
    labels = new_labels
    # UPDATE: centroid = ค่าเฉลี่ยสมาชิก
    for c in range(k):
        m = Z[labels == c]
        if len(m): C[c] = m.mean(0)

5 เลือก k เท่าไหร่?

k-means บังคับแบ่ง k กลุ่มเสมอ ต่อให้ข้อมูลไม่มีกลุ่มจริง — ต้องมีเครื่องมือช่วยตัดสิน

Elbow (ดู inertia)

เพิ่ม k → inertia ลดเสมอ หาจุดที่กราฟ "หักศอก" ลดต่อแล้วได้กำไรน้อย

Silhouette (แนะนำ)

เลือก k ที่ให้ค่าสูงสุด — วัดการแยกกลุ่มโดยตรง อ่านง่ายกว่า elbow

silhouetteตีความ
> 0.50กลุ่มชัด แยกขาดจริง
0.35 – 0.50พอใช้ — มีโครงสร้าง ขอบเบลอ
0.25 – 0.35อ่อน — เริ่มคาบเกี่ยว
< 0.25แทบไม่มีกลุ่มจริง
ผลรันจริงจากไฟล์ (ซ่อน 4 ก้อน): silhouette เลือก k=4 ได้ถูก (0.747) และ inertia หักศอกที่ k=4 พอดี (273.9 → 58.4) — สองเครื่องมือเห็นตรงกัน

6 กับดักที่พบบ่อย

ลืม z-score → feature สเกลใหญ่กลบตัวอื่น กลุ่มกลายเป็น "เรียงตาม feature เดียว" โดยไม่รู้ตัว (บั๊กเงียบ ไม่มี error)
สุ่ม init มั่ว → ติด local minimum, centroid ซ้อนกัน แก้ด้วย k-means++ + หลาย restart
เดา k → k-means แบ่งให้ครบเสมอ ต้องยืนยันด้วย silhouette/elbow ไม่ใช่ความรู้สึก
ข้อมูลน้อยเกิน → n ต่อกลุ่มน้อย สถิติไม่นิ่ง silhouette เชื่อไม่ได้
กลุ่มไม่กลม / ขนาดต่างมาก → k-means ชอบกลุ่มทรงกลมขนาดใกล้กัน ถ้ารูปร่างแปลก ใช้ DBSCAN / GMM แทน

7 แบบฝึกหัด (ทดลองในไฟล์)

ข้อ 1 · พิสูจน์ว่า z-score จำเป็น
เอา standardize() ออก (ใช้ X ดิบ) แล้วขยาย feature หนึ่งให้สเกลใหญ่ (×1000) → ดูว่ากลุ่มเพี้ยนเป็น "เรียงตาม feature นั้น" ยังไง
ข้อ 2 · เจอ local minimum ด้วยตา
ตั้ง restarts=1 แล้วรันซ้ำ 20 ครั้ง → จด inertia ทุกครั้ง จะเห็นบางครั้งกระโดดสูง = ติดหลุมพราง
ข้อ 3 · silhouette หา k ผิดเมื่อไหร่
ลด spread ให้ก้อนซ้อนกัน → ดูว่า silhouette ยังเลือก k ถูกไหม ที่จุดไหนเริ่มพัง
ข้อ 4 · ข้อมูลจริง
เปลี่ยน make_blobs() เป็น dataset จริง (Iris / Mall Customers) → เทียบ label ที่ได้กับหมวดจริง
ข้อ 5 · เทียบกับ library
รัน sklearn.cluster.KMeans ข้อมูลเดียวกัน → เทียบ inertia กับของเรา ควรใกล้กันมาก

8 อภิธานศัพท์

centroid
จุดศูนย์กลางของกลุ่ม = ค่าเฉลี่ยของสมาชิก
inertia (J)
ผลรวมระยะ² จากทุกจุดถึง centroid ของมัน ยิ่งต่ำยิ่งกระชับ
converge
ภาวะที่วนต่อแล้วไม่มีจุดไหนย้ายกลุ่ม = อัลกอริทึมหยุด
local minimum
คำตอบที่ "ดีในละแวก" แต่ไม่ใช่ดีที่สุดทั้งหมด เกิดจาก init ไม่ดี
k-means++
วิธี init ที่เลือก centroid เริ่มให้กระจายตัว ลดโอกาสติด local minimum
silhouette
คะแนน −1..1 บอกว่ากลุ่มแยกขาดกันดีแค่ไหน
elbow
จุดหักศอกบนกราฟ inertia-vs-k ใช้ช่วยเลือก k
